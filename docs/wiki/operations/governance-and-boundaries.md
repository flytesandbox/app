# Governance and Boundaries

## Ecosystem rule

- The ecosystem is not constrained by a retired launch sequence.
- Operational knowledge belongs in topic-based Wiki and `Support` surfaces.

## Compliance-minded boundaries

- Environment separation, least privilege, secrets handling, and auditability
  remain active control points.
- Documentation must describe the current system, not the temporary tooling
  that originally assembled it.

## Maintenance rule

- New support or maintenance work should update the relevant topic-based docs
  when a stable rule changes.
- Archive outputs may preserve evidence, but active guidance must stay concise,
  current, and ecosystem-specific.
